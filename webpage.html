iomanip

The iomanip library in C++ is used to manipulate the format of input and output streams. It provides a set of manipulators that allow you to control the appearance of data when it is displayed or read. Here are some of the common uses of iomanip


Setting Field Width

The setw manipulator is used to set the minimum width of the next output field. This is useful for aligning data in columns and creating well-organized output.

cout << setw(8) << num1 << setw(8) << num2 << setw(8) << num3 << endl;

Setting Precision

The setprecision manipulator is used to set the precision or the number of decimal places to be displayed when outputting a floating-point number.

cout << setprecision (3) << pi << endl; // Output: 3.14

Setting Fill Character

The setfill manipulator is used to change the fill character used to pad the output to the specified width .

cout << setfill('0') << setw(6) << num << endl; // Output: 000042

Setting Base

The setbase manipulator is used to set the base for integer output, allowing you to display numbers in decimal, hexadecimal, or octal format.

cout << setbase(16) << num << endl; // Output: A7F

Formatting monetary values

the'get money'and'put money' manipulators are used to parse and format monetary values resspectively


Formatting date and time

the'get time'and'put time' manipulators are used to parse and format date and time according to a specified format.
By using the manipulators provided by the 'iomanip' library, you can create more readable and visually appealing output in your C++ programs.


Cpp File Extensions :-

.pdb: This file extension stands for "Program Database" and is used by Microsoft Visual Studio to store debugging information for a program. The .pdb file contains information about the program's symbols (e.g., function names, variable names, line numbers), which can be used to debug the program more effectively. When you compile a program in Visual Studio with debugging information enabled, a .pdb file is generated alongside the executable file (.exe).

.obj: This file extension stands for "Object File" and is used to store compiled code that has not yet been linked with other object files or libraries. When you compile a program, the compiler generates one or more .obj files, which contain the machine code for the individual functions and variables in the program. These .obj files are then linked together by the linker to create the final executable file (.exe).

.ilk: This file extension stands for "Incremental Linking File" and is used by Microsoft Visual Studio to store information about the linking process. The .ilk file contains information about the dependencies between the various object files and libraries that are linked together to create the final executable file (.exe). This information is used to speed up the linking process by only re-linking the parts of the program that have changed since the last build.

.exe: This file extension stands for "Executable File" and is used to denote a file that can be executed directly by the operating system. The .exe file contains the machine code for the program, as well as any necessary resources (e.g., images, sounds, data files) that are required for the program to run. When you double-click on a .exe file, the operating system loads the file into memory and begins executing the program.

pointers uses :

A pointer is a variable that stores the memory address of another variable. Pointers are useful for several reasons:
1. Memory Efficiency: Pointers allow us to pass large data structures to functions without making a copy of the data. This can save memory and improve performance.
2. Dynamic Memory Allocation: Pointers enable us to allocate memory dynamically during runtime, which is useful when the amount of memory needed is not known in advance.
3. Code Organization: Pointers can help make code more organized and modular by allowing functions to modify their arguments directly, rather than requiring the function to return a modified copy of the data.
4. Complex Data Structures: Pointers are essential for implementing complex data structures like linked lists, trees, and graphs, which are used in many real-world applications. Overall, pointers provide flexibility, efficiency, and power to the programmer, but they also require careful handling to avoid errors like null pointer dereferencing or memory leaks.

Abstract classes

Abstract classes in C++ are a fundamental concept in object-oriented programming. They are used to define a blueprint for derived classes to follow, providing a common interface for them to implement.   An abstract class in C++ is a class that is designed to be specifically used as a base class. It contains at least one pure virtual function, which is a function that must be implemented by any derived class. Abstract classes cannot be instantiated directly, but pointers and references to abstract class types can be created.

conio.h

conio.h is a C header file that provides functions for console input/output operations. It is commonly used in C programming for developing interactive console applications.

conio.h Functions

Functions in conio.h

Some of the key functions provided by conio.h include:

  • kbhit: Determines if a keyboard key was pressed.
  • cgets: Reads a string directly from the console.
  • cscanf: Reads formatted values directly from the console.
  • putch: Writes a character directly to the console.
  • cputs: Writes a string directly to the console.
  • cprintf: Formats values and writes them directly to the console.
  • clrscr: Clears the screen.
  • getch: Gets a character from the console without echoing it.
  • getche: Gets a character from the console and echoes it.
  • gotoxy: Moves the cursor to a specific position on the screen.
  • wherex: Returns the current x-coordinate of the cursor.
  • wherey: Returns the current y-coordinate of the cursor.
  • highvideo: Brightens a character string and displays it on the screen.
  • movetext: Moves text on the screen.
  • getpass: Takes input from the user without displaying it on the screen.
